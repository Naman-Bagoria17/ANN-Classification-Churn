--------TensorFlow and Keras--------
When you start working with Artificial Neural Networks (ANNs), you quickly realize that building them from scratch involves heavy matrix multiplications, gradient calculations using backpropagation, optimization of weights, and handling large datasets efficiently. Doing all of this manually in C++ or even Python is very complex, error-prone, and time-consuming. That is why we use tools like TensorFlow and Keras. TensorFlow is a low-level deep learning framework developed by Google that acts as the computational engine; it handles tensor operations, automatic differentiation for gradient computation, efficient execution on CPU/GPU, and scaling to large real-world models. On the other hand, Keras is a high-level API (now integrated with TensorFlow) that provides a simple and user-friendly way to build neural networks without manually implementing forward and backward propagation. In simple terms, TensorFlow does the actual computation while Keras provides an easy interface to design models. They work together such that you define the model using Keras, and TensorFlow executes the underlying computations. A good analogy is that Keras is like the driver and TensorFlow is like the car engine—you don’t need to understand the engine in detail to drive the car. This is why your sir asked you to learn them, because in ANN projects you need to build models quickly, experiment with different architectures, train efficiently, and focus on concepts instead of coding complex mathematics from scratch. In short, TensorFlow and Keras simplify the process of building and training neural networks by combining computational power with ease of use.


---ANN is sequential in structure not in time---
ANN (Artificial Neural Network) is called a sequential network mainly because of how data flows through it. In a standard feedforward ANN, the computation happens layer by layer in a fixed sequence: Input layer → Hidden layer(s) → Output layer. Each layer takes input from the previous layer only and there is a strict forward direction (no cycles, no skipping backward). So the processing follows a sequential pipeline, where input features are fed into the first layer, that layer computes outputs using weights and activation, these outputs become inputs to the next layer, and this continues until the final output is produced. Mathematically, it looks like Output = f3(f2(f1(X))). This nested function structure itself is sequential since each function depends on the result of the previous one. The core idea is that layers are ordered, computation is step-by-step, there is no parallel time dependency like in Recurrent Neural Networks, and no branching or skip connections in a basic ANN. ANN is not sequential in time (that is a property of Recurrent Neural Network), it is sequential in structure through layer-wise computation.
RNN has parallel time dependency because at each time step, the current output depends not only on the current input but also on the previous hidden state, which carries information from earlier time steps. This creates a chain of dependencies across time, where each step is linked to past computations, allowing the network to capture sequential patterns. However, due to this dependency on previous states, computations cannot be fully parallelized across time steps, unlike ANN, and must follow a temporal sequence as in Recurrent Neural Network.
